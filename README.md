This repo contains the latest computer-vision-based beam characterization code from NLR (NREL). The most recent code is in the "BCS_2026_01_30.ipynb" file

The overall workflow is shown in the figure below.
![Workflow](readme_figure_1.png)

There are four main steps in the workflow:
1. **Cropping and rectification**: The goal of this step is to obtain an image that contains only the BCS target. Cropping uses a user-defined corner locations [ul, ur, lr, ll] to crop the image. Rectification uses the same corner locations to perform a perspective transform to obtain a rectified image of the BCS target. The rectified image dimension is arbitrary, as the final step will be converted to the original image dimension, so the rectified image dimension is set to 1000x1000 pixels for simplicity.
2. **Background removal**: The goal of this step is to remove the non-black pixels in the rectified image such that they do not contribute to the centroid caluclation code. The background removal uses a histogram-based method to determine the threshold for background removal. It asserts that the most frequent pixel value is in the background, and uses a slightly higher pixel value (1.2x) as the threshold for background removal. A soft threshold is used to create a more natural transition between the background and the beam, which can help improve the accuracy of the centroid calculation.
3. **Centroid calculation**: The goal of this step is to calculate the centroid of the beam in the rectified image. The centroid is calculated using the center of mass method, which calculates the weighted average of the pixel coordinates, where the weights are the pixel values. The centroid is then converted back to the original image coordinates using the inverse perspective transform.
4. **Coordinate revert back**: The goal of this step is to convert the centroid coordinates from the rectified image back to the original image coordinates. This is done using the inverse perspective transform, which uses the same corner locations as the cropping and rectification step to perform the transformation. The resulting centroid coordinates are in the original image coordinate system, which can be used for further analysis or visualization.
